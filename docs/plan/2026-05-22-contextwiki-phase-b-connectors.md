## Acceptance Criteria

- `SourceRegistry` can include GitHub and Web sources without breaking Notion/Tistory.
- `list_sources()` exposes `source_github` and `source_web` with `auth_ref` values that reference environment variables, not raw secrets.
- `sync_source("source_github")` can fetch configured repositories, produce `DocumentModel` records with:
  - stable `external_id` and `document_id`,
  - GitHub canonical blob URLs,
  - file paths,
  - blob SHA or equivalent `version_id`,
  - source id/type metadata.
- GitHub ingestion filters to bounded text/code/markdown files and fails the sync on fetch errors so missing files are not tombstoned after a partial connector failure.
- `sync_source("source_web")` can fetch configured docs URLs, parse sitemap entries or bounded same-origin links, respect robots.txt disallow rules, extract readable text/title/canonical URLs, and produce stable URL-based identities.
- Website/docs ingestion fails the sync on required fetch errors and only tombstones missing documents after a successful bounded crawl.
- Phase B connector output flows through existing source-aware chunking so code files get path and line-range citations and web/markdown docs get citation-ready metadata.
- Tests cover mocked GitHub ingestion, mocked website/docs crawling, source registration, disabled-source behavior, and integration through `IngestionService`.
- README documents configuration environment variables and source ids.

## Architecture/ADR Constraints

- ADR 0001 keeps external API behavior in `fetching/`, indexing/chunk writes in `indexing/`, retrieval in `search/`, and MCP formatting in `api/`.
- ADR 0002 keeps SQLite metadata as the source/job/document/chunk authority and Chroma as vector retrieval.
- ADR 0003 requires stable document identity, `canonical_url`, `last_seen_at`, `last_seen_sync_id`, `deleted_at`, `version_id`, line-range code chunks, and source-wide stale cleanup only after successful full source syncs.
- Connector auth must use environment-variable references in source metadata. Raw tokens must not be stored in SQLite or docs.
- Tests must use mocked HTTP/API responses and temporary stores; they must not touch user Chroma data or live external services.

## Risks and Rollback Notes

- Risk: Partial GitHub/Web fetch failures could tombstone still-valid documents.
  - Mitigation: connector fetch errors fail the sync; stale cleanup only happens after a successful complete bounded snapshot.
- Risk: Broad website crawling could become unbounded or disrespect robots policy.
  - Mitigation: enforce max pages, same-origin crawling, optional sitemap parsing, robots disallow checks, and a configurable crawl delay.
- Risk: GitHub repositories can be large or contain binary/secrets files.
  - Mitigation: enforce file extension/text filters, max file count, max file byte size, and do not log file contents or token values.
- Risk: Adding new sources could change existing Notion/Tistory behavior.
  - Mitigation: register new connectors additively and keep existing MCP tool contracts stable.
- Rollback: revert this branch; no user Chroma data is reset or migrated.
